Набор скиллов для code review в [Claude Code](https://docs.anthropic.com/en/docs/claude-code) и [Codex CLI](https://github.com/openai/codex).

Эти скиллы не просто запускают линтер — они проверяют код на соответствие **правилам вашего проекта**: кодстайл, архитектура, именование, допустимые паттерны, проектные решения. Каждое ревью параллельно запускает 4 специализированных подагента, покрывающих разные домены. Результат — структурированный отчёт с конкретными, применимыми предложениями по исправлению.

> **Качество ревью ровно настолько хорошо, насколько хороши правила вашего проекта.** Скиллы проверяют код на соответствие конвенциям, которые вы описали в файлах правил. Хорошо написанные, конкретные правила (кодстайл, архитектура, именование, запрещённые паттерны) дают детальные, полезные ревью. Расплывчатые или отсутствующие правила приводят к поверхностным, общим замечаниям. Вложите время в проработку правил — именно это делает инструмент мощным.

**Откуда читаются правила:**

- **Claude Code**: `.claude/rules/*.md`, `CLAUDE.md`
- **Codex CLI**: `AGENTS.md` (корневой и вложенные)

## Скиллы

### `/code-review`

Параллельный диспетчер ревью. Одновременно запускает 4 специализированных подагента, каждый проверяет код по собственному чеклисту:

- **Codestyle** — именование, форматирование, запрещённые паттерны, конвенции проекта
- **Architecture** — SOLID, слоистая архитектура, инверсия зависимостей, разделение ответственности
- **Security** — OWASP top 10, валидация входных данных, авторизация, утечка данных
- **Testing** — покрытие, паттерн AAA, граничные случаи, качество тестов

Поддерживаемые области проверки:

| Флаг | Область |
|------|---------|
| *(без флага)* | Закоммиченный diff: текущая ветка vs main |
| `--uncommitted` | Staged + unstaged + untracked изменения |
| `--pr [number]` | Diff pull request'а |
| `--all` | Все файлы проекта |
| `--plan <path>` | Проверка дизайн-документа |
| `--impl <path>` | Проверка кода, реализующего план |
| `<paths>` | Конкретные файлы или директории |

Понимает естественный язык — не нужно запоминать флаги:

| Вы пишете | Что произойдёт |
|-----------|----------------|
| `/code-review` | Закоммиченный diff vs main |
| `/code-review незакоммиченный код` | Незакоммиченные изменения |
| `/code-review проверь PR` | Diff PR текущей ветки |
| `/code-review проверь план docs/plans/auth.md` | Проверит дизайн-документ |
| `/code-review apps/api/` | Проверит файлы в директории |

Результат: структурированный Markdown-отчёт, сгруппированный по критичности (Critical > Important > Minor), с конкретными предложениями по исправлению каждой найденной проблемы.

### `/code-review-fix`

Интерактивный процессор исправлений по отчёту code review. Проходит по каждой проблеме:

1. Показывает проблему с контекстом
2. Предлагает варианты исправления с плюсами и минусами
3. Пользователь выбирает вариант
4. Запускает подагента для применения исправления
5. Обновляет статус в файле отчёта (DONE / SKIP / CHANGED)

Может обработать отчёт из файла или из контекста разговора сразу после `/code-review`.

## Установка

### Быстрая установка (из последнего релиза)

```bash
# Claude Code
curl -fsSL https://raw.githubusercontent.com/unger1984/codepatrol/main/install-remote.sh | bash

# Codex CLI
curl -fsSL https://raw.githubusercontent.com/unger1984/codepatrol/main/install-remote.sh | bash -s codex
```

### Claude Code (как плагин через marketplace)

```
/plugin marketplace add unger1984/codepatrol
/plugin install codepatrol@unger1984/codepatrol
```

Скиллы будут доступны как `/codepatrol:code-review` и `/codepatrol:code-review-fix`.

### Из исходников

```bash
git clone https://github.com/unger1984/codepatrol.git
cd codepatrol
./install.sh claude  # или: ./install.sh codex
```

## Требования

- **git** — необходим для diff-based режимов проверки
- **gh** (GitHub CLI) — необходим для флага `--pr`
- Внешние MCP-серверы или плагины не требуются — скиллы используют только встроенные инструменты агентов

## Разработка

Репозиторий использует шаблонный подход для поддержания единого источника истины для обеих платформ.

```
templates/                        # Исходные шаблоны с {{PLACEHOLDER}}-ами
platforms/                        # Файлы переменных для каждой платформы
plugins/codepatrol/               # Плагин (устанавливается через marketplace)
  skills/                         # Сгенерированные файлы скиллов (plugins/codepatrol/skills/)
  .claude-plugin/plugin.json      # Манифест плагина
.claude-plugin/marketplace.json   # Манифест marketplace
install.sh                        # Скрипт сборки и установки
```

```bash
./install.sh build   # Пересобрать скиллы плагина из шаблонов
./install.sh codex   # Сгенерировать и установить для Codex
./install.sh claude  # Сгенерировать и установить локально для Claude
```

## Известные ограничения

- Скиллы тестировались только на **macOS**. На Linux и Windows могут работать, но не проверялись — контрибьюции и баг-репорты приветствуются.
- **Cursor** пока не поддерживается. Формат скиллов и механизм диспетчеризации агентов нужно адаптировать под архитектуру Cursor.
